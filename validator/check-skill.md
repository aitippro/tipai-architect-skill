# Skill 格式校验清单

Skill 文件变更后，逐项验证。不通过的项目标记为 `⚠ 需修改`。

> 适用于 `skills/` 目录下所有 Skill 文件。

## 结构完整性
- [ ] 文件开头有 `# Skill:` 标题
- [ ] 标题后有 Skill 定位的一句话描述
- [ ] 明确声明了角色数量与名称（至少 3 个 AI 角色）
- [ ] 每个角色有：名称、职责、风格
- [ ] 工作流步骤完整且有编号（Step 1 → Step N）

## 角色定义
- [ ] 角色分工无重叠（各有独立职责域）
- [ ] 每个角色有名称（非空，>2 字符）
- [ ] 角色风格描述具体（不含"负责一切""处理所有"等空洞词）

## 工作流
- [ ] 步骤有清晰的输入/输出
- [ ] 步骤间有明确的触发条件（何时进入下一步）
- [ ] 至少包含：需求采集 / 方案设计 / 质量检验 / 输出生成 四大环节之一
- [ ] 每步给出了操作指引（不是只有标题）

## 输出规范
- [ ] 明确了最终输出格式
- [ ] 引用了模板文件（如有配套 templates/）
- [ ] 模板路径指向存在的文件

## 质量守则
- [ ] 包含至少一条防空洞规则（禁止虚构数据/无效论据）
- [ ] 包含至少一条建设性规则（质疑必须带替代方案）
- [ ] 包含至少一条进度控制规则（防无限循环/原地打转）

## 示例与引用
- [ ] 引用了至少一个完整示例（如有 examples/）
- [ ] 示例路径指向存在的文件
- [ ] 给出了用户快速触发命令

## 兼容性
- [ ] Skill 文件名与 Skill 标题一致
- [ ] 未硬编码本地路径 / IP / 主机名 / 凭证
- [ ] 引用的外部 URL 使用通用域名（非 localhost / 内网地址）
