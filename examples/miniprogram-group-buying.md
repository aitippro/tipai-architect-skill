# 示例: 微信小程序社区团购

## 用户输入
> "我想在三四线城市做一个社区团购小程序，主打本地生鲜次日达"

## Step 1-2: 裁判追问 + 方案设计

```
Q1: 目标城市规模？
A1: 50-100 万人口地级市，先从老家(湖北孝感)试点

Q2: 供应链怎么解决？
A2: 本地农贸市场批发+周边农户直供，自己不做仓储

Q3: 团长怎么招募？
A3: 宝妈+便利店店主，佣金 10-15%

Q4: 技术栈偏好？
A4: 微信小程序原生 + 云开发(CloudBase)

Q5: 预算和时间？
A5: 3 万 RMB，1.5 个月上线
```

## 初版方案 (摘要)

**方案: 邻里鲜 v1.0**

- **问题**: 三四线城市生鲜电商渗透率低，传统团购靠微信群人工接龙
- **方案**: 微信小程序+云开发，团长端+用户端双端，次日达
- **竞品**: 美团优选(大平台)/兴盛优选(加盟/门槛高)/微信群接龙(无系统)
- **技术**: 原生小程序 / CloudBase(数据库+云函数) / 微信支付 / 订阅消息
- **盈利**: 商品差价 15-20% + 团长佣金 12%

## Step 4: 答辩片段

```
第3轮 — 盈利性
✅ 正方: 生鲜毛利 25%，扣除团长佣金 12%+物流 8%，净利 5%
❌ 反方: 美团优选在孝感已有 200+ 团长，你的价格能比美团低吗？
       建议: 不做标准生鲜(鸡蛋/蔬菜)，做地方特产(孝感麻糖/米酒)
✅ 正方: 同意差异化。MVP 30% SKU 为地方特产，70% 高频生鲜引流

第6轮 — 竞争壁垒
✅ 正方: 本地农户直供关系需要时间建立，能形成供应链壁垒
❌ 反方: 农户关系 3 个月可被复制。真正的壁垒是团长网络(人)。
       建议: 团长分级(金牌/银牌/铜牌)，不同佣金+激励
👨‍⚖️ 裁判: 正方 7/10 反方 7/10。采纳团长分级
```

## Step 5: 裁决

```
🏆 判定: 🟡 有条件通过

修改点:
1. 30% SKU 为地方特产(差异化)
2. 团长分级体系(金牌 15%/银牌 12%/铜牌 10%)
3. 首批只覆盖 5 个小区，验证模型再扩

正方 7.0/10  反方 7.5/10  达成共识 75%
```

## Step 6: 任务拆解

| ID | 任务 | 依赖 | 验收 |
|----|------|------|------|
| T001 | 小程序注册+CloudBase 初始化 | 无 | 开发者工具编译通过 |
| T002 | 微信登录+手机号绑定 | T001 | 获取 unionId+手机号 |
| T003 | 商品管理(CRUD+分类+图片) | T001 | 后台发布→前台展示 |
| T004 | 购物车+下单 | T002+T003 | 选品→加购→下单 |
| T005 | 微信支付集成 | T004 | 真实支付→回调通知 |
| T006 | 团长端(订单管理+佣金) | T005 | 团长看订单→核销→佣金 |
| T007 | 订阅消息(到货提醒) | T005 | 订单状态变更推送 |
