# 示例: SaaS 客服工单系统

## 用户输入
> "我想做一个面向中小团队的 SaaS 客服工单系统，支持多渠道接入"

## Step 1-2: 裁判追问 + 方案设计

```
Q1: 目标客户规模和付费能力？
A1: 10-50 人团队，$29-99/月可接受

Q2: 竞品了解吗？
A2: Zendesk(太贵太重)、Freshdesk(功能多但复杂)、飞书客服(国内生态绑定)

Q3: 技术栈偏好？
A3: Next.js 全栈，PostgreSQL，部署在 Vercel/AWS

Q4: 预算和时间？
A4: 2 人 4 个月出 MVP

Q5: 核心差异点？
A5: AI 自动分类+建议回复+知识库自动沉淀
```

## 初版方案 (摘要)

**方案: HelpFlow SaaS v1.0**

- **问题**: 中小企业用不起 Zendesk($49/人/月起)，开源方案维护成本高
- **方案**: Next.js 全栈 SaaS，AI 驱动的工单分类+自动回复建议
- **竞品**: Zendesk(贵)/Freshdesk(臃肿)/飞书客服(生态锁定)
- **技术**: Next.js 15 / Prisma+PostgreSQL / tRPC / OpenAI API / Stripe
- **盈利**: $29/月起，3 人免费

## Step 4: 答辩片段

```
第2轮 — 技术可行性
✅ 正方: Next.js 全栈已被 Vercel/Netlify 验证；AI 回复建议用 GPT-4o-mini 延迟 <2s
❌ 反方: 多租户数据隔离复杂，PostgreSQL Row-Level Security 有性能坑。
       建议: 用 schema-per-tenant 方案，先做单租户 MVP
✅ 正方: 接受。MVP 阶段单 DB + tenant_id 隔离，Pro 版再上 schema 隔离

第7轮 — 竞争壁垒
✅ 正方: AI 知识库自动沉淀是壁垒，越用越智能
❌ 反方: AI 功能竞品 3 个月可复制。真正的壁垒是数据迁移成本。
       建议: 先做到 50 个付费客户，验证 PMF 再谈壁垒
👨‍⚖️ 裁判: 正方 6/10 反方 8/10。修正：MVP 专注工单核心体验，AI 功能 Phase 2
```

## Step 5: 裁决

```
🏆 判定: 🟡 有条件通过

修改点:
1. MVP 单租户 tenant_id 隔离
2. AI 功能延迟到 Phase 2
3. 定价先 $29/月 单一套餐，验证后再分 tier

正方 7.0/10  反方 7.5/10  达成共识 75%
```

## Step 6: 任务拆解

| ID | 任务 | 依赖 | 验收 |
|----|------|------|------|
| T001 | Next.js 脚手架 + Prisma + tRPC | 无 | 页面渲染，DB 连接成功 |
| T002 | 用户注册/登录 (NextAuth) | T001 | 邮箱+Google 登录 |
| T003 | 工作区创建+成员邀请 | T002 | 创建+邀请+权限 |
| T004 | 工单 CRUD + 状态流转 | T001 | 创建→处理中→已解决 |
| T005 | 邮件渠道接入 (Inbound) | T004 | 发邮件自动创建工单 |
| T006 | Stripe 订阅+账单 | T002 | 月付+年付，Webhook |
| T007 | 知识库基础 (手动创建) | T001 | 创建/编辑/搜索文章 |
