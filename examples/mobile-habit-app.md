# 示例: 移动端习惯打卡 App

## 用户输入
> "我想做一个极简的习惯打卡 App，主打社交激励，用户之间可以互相监督"

## Step 1-2: 裁判追问 + 方案设计

```
Q1: 目标用户是谁？
A1: 大学生和职场新人，18-30 岁，想培养好习惯但缺乏动力

Q2: 为什么不用现有的？
A2: Streaks 太贵($4.99/月)，Habitica 太游戏化，小日常 功能太简单

Q3: 技术栈偏好？
A3: Flutter 跨平台(iOS+Android)，Firebase 后端

Q4: 预算和时间？
A4: 独立开发，2 个月 MVP

Q5: 盈利方式？
A5: 免费 + 内购去广告，社交功能免费
```

## 初版方案 (摘要)

**方案: HabitBuddy v1.0**

- **问题**: 单人打卡缺乏持续动力，现有工具偏自我管理
- **方案**: Flutter 跨平台，好友互相关注+打卡可见+轻量互动
- **竞品**: Streaks(贵)/Habitica(太重)/小日常(无社交)/Loop Habit Tracker(仅 Android)
- **技术**: Flutter 3.x / Firebase Auth+Firestore / FCM 推送 / RevenueCat 内购
- **盈利**: 免费 + $2.99 去广告

## Step 4: 答辩片段

```
第3轮 — 盈利性
✅ 正方: 参考 Waterllama 等 App，$2.99 一次买断+少量广告可盈亏平衡
❌ 反方: 社交 App 需要网络效应。100 DAU 时社交激励几乎为 0。
       建议: 先做单机打卡验证留存，再上线社交功能
✅ 正方: 部分接受。MVP 做单机打卡 + 好友列表(非实时)，社交动态 Phase 2

第7轮 — 用户体验
✅ 正方: 每天打开 <5 秒完成打卡，交互极简
❌ 反方: 打卡 App 的留存率普遍 <30%(7日)。用户不是缺工具，是缺动力。
       建议: 加 Widget 桌面小组件，让打卡在解锁时可见
👨‍⚖️ 裁判: 正方 7/10 反方 7/10。采纳 Widget 建议
```

## Step 5: 裁决

```
🏆 判定: 🟡 有条件通过

修改点:
1. MVP 不包含社交动态(保留好友列表)
2. 必须包含桌面 Widget
3. 盈利推迟到 1000 DAU 后测试

正方 7.0/10  反方 7.5/10  达成共识 80%
```

## Step 6: 任务拆解

| ID | 任务 | 依赖 | 验收 |
|----|------|------|------|
| T001 | Flutter 项目+Firebase 配置 | 无 | iOS+Android 编译通过 |
| T002 | 邮箱注册+登录 | T001 | 注册→登录→登出 |
| T003 | 习惯 CRUD (名称/图标/频次) | T002 | 创建/编辑/删除/列表 |
| T004 | 每日打卡+连续天数 | T003 | 打卡动画+连续计数 |
| T005 | 好友搜索+关注 | T002 | 搜索→发送请求→接受 |
| T006 | 好友打卡列表(只读) | T004+T005 | 看到好友今日打卡 |
| T007 | Widget 桌面小组件 | T003 | 今日待打卡习惯+一键签到 |
