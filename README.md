# 🏛️ TipAI Architect Skill

**三方 AI 答辩式方案架构师**

用户提需求 → 裁判设计+调研 → 正方 vs 反方辩论 → 裁决 → 拆解为原子任务

[![License](https://img.shields.io/badge/License-NonCommercial-red?style=flat-square)](LICENSE)

</div>

---

## 🎯 核心机制

一个 Skill 文件，Agent 加载后自动扮演 **3 个 AI 角色**：

| 角色 | 职责 |
|------|------|
| 👨‍⚖️ **裁判** | 主持全流程、调研市场、出方案、裁决赛果、拆解任务 |
| ✅ **正方** | 为方案辩护、提供证据、回应质疑 |
| ❌ **反方** | 寻找漏洞、质疑可行性/盈利性/技术风险 |

---

## 📖 工作流

```
需求采集(裁判追问5轮+) → 调研设计(竞品+技术+市场) → 方案预览(用户审阅)
    → 三方答辩(5-30轮) → 最终裁决 → 任务拆解
```

---

## 🚀 使用

1. 加载 `skills/solution-forge.md` 到你的 Agent
2. 告诉 Agent: "我有一个想法，帮我做方案设计"

```
用户: "我想做一个AI驱动的本地优先的提示词工程桌面工具"

Agent(裁判): "好的。让我先了解你的需求。
第一问：你的核心目标是什么？解决谁的什么问题？"

用户: "帮助写作者和开发者生成更好的AI提示词"

Agent(裁判): "第二问：目标用户规模多大？你预期他们愿意付费吗？"

... (追问5轮后)

Agent(裁判): "[产出初版方案 + 竞品调研 + 技术建议]
是否认可这个方向？认可后我们进入答辩环节。"

用户: "进入答辩"

Agent(裁判): "现在开庭。辩题：本方案是否值得投入开发？
正方先发言。"

Agent(正方): "我支持。理由：1) 提示词工程市场年增长40%..."
Agent(反方): "我对增长数据有质疑。实际付费转化率..."
... (辩论10轮)

Agent(裁判): "[裁决报告] 判定：通过。
现将方案拆解为开发任务：[23个原子任务清单]"
```
